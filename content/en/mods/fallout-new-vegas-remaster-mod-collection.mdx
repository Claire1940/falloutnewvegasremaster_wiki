<Section icon={Shield} title="Performance and Core Stability Essentials">

Before deploying heavy visual mods, you must secure the game engine. Fallout: New Vegas was released in 2010 and is inherently limited by older memory management systems. Modern systems require specific engine-level patches to utilize contemporary hardware effectively.

The stability requirements differ slightly depending on which digital storefront you purchased the game from. GOG users enjoy a pre-patched executable, while Steam users must apply manual adjustments.

| Feature | Steam Version | GOG Version |
| :--- | :--- | :--- |
| **4GB Patch** | Required (Manual installation) | Pre-Patched out of the box |
| **Mod Compatibility** | High (Requires NVSE) | High (Requires NVSE) |
| **Out-of-box Stability** | Moderate | High |
| **Setup Difficulty** | Low (Requires patcher run) | Very Low |

To prepare your game for Gopher's visual remaster collection, follow this foundational setup process.

<Steps>
  <Step num="1" title="Clean Installation">
    Install Fallout: New Vegas through Steam or GOG. Ensure the game directory is located outside of protected system folders like Program Files to avoid write-permission errors.
  </Step>
  <Step num="2" title="Apply 4GB Patcher">
    For Steam users, download the Fallout 4GB Patcher from Nexus Mods. Place the executable in your game folder and run it once. This allows the game to access 4GB of virtual memory instead of 2GB.
  </Step>
  <Step num="3" title="Install Stability Base">
    Download and deploy a dedicated stability collection. This should include NVSE (New Vegas Script Extender), New Vegas Anti-Crash (NVAC), and New Vegas Tick Fix (NVTF) to eliminate micro-stuttering.
  </Step>
</Steps>

<Callout type="warning" title="Critical Patcher Requirement">
For Steam users, the 4GB Patcher is non-negotiable. Skipping this step will result in immediate out-of-memory crashes when loading high-resolution textures in heavily populated areas like the New Vegas Strip.
</Callout>
