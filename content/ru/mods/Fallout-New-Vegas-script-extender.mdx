Чтобы подготовить систему к работе с **расширителем скриптов Fallout New Vegas**, необходимо скачать и установить три основных зависимости. Они гарантируют, что ваша операционная система Windows сможет запускать пользовательский код, внедряемый расширителем.

| Инструмент | Цель | Тип источника |
| --- | --- | --- |
| **7-Zip** | Извлекает сжатые архивы модов .7z и .zip | Утилита |
| **Visual C++ Redistributable** | Предоставляет библиотеки среды выполнения, необходимые для современного xNVSE | Системная библиотека |
| **4GB Patch** | Позволяет игре использовать 4 ГБ оперативной памяти вместо 2 ГБ | Патч исполняемого файла |

Сначала скачайте и установите 7-Zip для работы с файлами. Затем скачайте последние библиотеки среды выполнения. Многие пользователи сталкиваются с моментальными вылетами из-за отсутствия необходимых библиотек Microsoft.

<Callout type="warning" title="Требование к системным библиотекам">
Вы должны установить [Microsoft Visual C++ Redistributable](https://learn.microsoft.com/en-us/cpp/windows/latest-supported-vc-redist) перед запуском расширителя скриптов. Без этого пакета загрузчик игры не сможет внедрить пользовательские библиотеки скриптов в процесс игры, что приведет к тихому вылету при запуске.
</Callout>

Наконец, скачайте утилиту 4GB Patch. Современные операционные системы ограничивают старые 32-битные приложения 2 ГБ виртуальной памяти. Патч модифицирует исполняемый файл игры для работы с 4 ГБ адресного пространства, что предотвращает вылеты из-за нехватки памяти при использовании текстур высокого разрешения или тяжелых скриптовых модов.

<Section icon={Target} title="Пошаговая настройка расширителя скриптов Fallout New Vegas">

Когда предварительные файлы будут готовы, можно переходить к ручной установке **расширителя скриптов Fallout New Vegas**. Процесс установки включает в себя размещение файлов непосредственно в корневом каталоге вашей игры, а не в папке Data, где находятся обычные моды.

<Steps>
  <Step num="1" title="Найдите директорию игры">
    Откройте лаунчер игры (например, Steam). Нажмите правой кнопкой мыши на игру в библиотеке, выберите «Управление» и нажмите «Просмотреть локальные файлы». Откроется корневая папка, содержащая основные исполняемые файлы игры.
  </Step>
  <Step num="2" title="Извлеките файлы NVSE">
    Откройте скачанный архив расширителя скриптов с помощью 7-Zip. Выберите все файлы внутри архива (в частности, загрузчик и dll-файлы) и перетащите их прямо в корневой каталог.
  </Step>
  <Step num="3" title="Примените патч 4 ГБ">
    Скачайте утилиту [FNV 4GB Patch](https://www.nexusmods.com/newvegas/mods/62552). Извлеките исполняемый файл в тот же корневой каталог. Нажмите правой кнопкой мыши на файл патча, выберите «Запуск от имени администратора» и нажмите любую клавишу, когда консоль подтвердит завершение патча.
  </Step>
  <Step num="4" title="Настройте лаунчеры">
    Патч 4 ГБ автоматически настраивает игру на загрузку расширителя скриптов при запуске стандартного исполняемого файла. Вам не нужно использовать старое приложение загрузчика для запуска игры.
  </Step>
</Steps>

Важно разместить файлы в правильной директории. Обратитесь к таблице структуры каталогов ниже, чтобы убедиться в правильности размещения файлов.

| Имя файла | Директория назначения | Тип файла |
| --- | --- | --- |
| **nvse_loader.exe** | Корневая папка (где находится FalloutNV.exe) | Исполняемый файл |
| **nvse_1_4.dll** | Корневая папка (где находится FalloutNV.exe) | Библиотека |
| **nvse_steam_loader.dll** | Корневая папка (где находится FalloutNV.exe) | Библиотека |
| **FNVPatch.exe** | Корневая папка (где находится FalloutNV.exe) | Исполняемый файл патчера |

<Callout type="info" title="Проверка директории">
Убедитесь, что файлы расширителя скриптов помещены в основную директорию, содержащую исполняемый файл игры, а не в папку Data. Размещение этих файлов внутри папки Data предотвратит загрузку расширителя скриптов.
</Callout>

Если вы используете версию игры от GOG, исполняемый файл уже пропатчен для поддержки 4 ГБ оперативной памяти «из коробки». Тем не менее, запуск ручной утилиты патчера все равно рекомендуется, чтобы гарантировать правильное подключение загрузчика расширителя скриптов к процедуре запуска.

</Section>

<Section icon={Sparkles} title="Проверка установки и настройка менеджера модов">

После копирования файлов вы должны убедиться, что расширитель скриптов корректно работает внутри игрового движка. Это можно проверить с помощью консоли разработчика. Кроме того, следует настроить менеджер модов для распознавания расширителя скриптов.

Чтобы проверить установку, запустите игру и нажмите клавишу тильда (~) на клавиатуре, чтобы открыть консоль разработчика. Введите команду `GetNVSEVersion` и нажмите Enter. Консоль должна вернуть номер активной версии. Если возвращается ошибка о том, что команда не найдена, значит файлы установлены в неправильную директорию.

Для управления модами рекомендуется использовать Mod Organizer 2. Он использует виртуальную файловую систему, которая сохраняет директорию вашей игры чистой, предотвращая необратимую перезапись базовых файлов игры модами.

<Callout type="tip" title="Настройка Mod Organizer 2">
При использовании Mod Organizer 2 всегда запускайте игру через опцию пропатченного исполняемого файла, настроенную в менеджере. Убедитесь, что путь к программе указывает непосредственно на пропатченный исполняемый файл в корневом каталоге для загрузки всех виртуализированных модов и расширений скриптов.
</Callout>

Если вы решите обновить расширитель скриптов в будущем, скачайте последний релиз из [репозитория xNVSE на Github](https://github.com/xNVSE/xnvse) и перезапишите существующие файлы в корневом каталоге. Это обновит библиотеки, не затрагивая установленные моды или файлы сохранений.

<Section icon={Swords} title="Устранение распространенных ошибок NVSE">

Даже при тщательной установке конфликты конфигурации или отсутствие обновлений системы могут вызвать ошибки. Понимание того, как диагностировать эти проблемы, помогает поддерживать стабильность игры.

| Симптом ошибки | Общая причина | Решение |
| --- | --- | --- |
| **Вылет при запуске** | Отсутствуют Visual C++ Redistributables | Установите последние версии сред выполнения x86 и x64 |
| **Команда не найдена** | Файлы помещены в неправильную папку | Переместите файлы из Data в корневую папку |
| **Патчер не запускается** | Ограничены права администратора | Нажмите правой кнопкой на патчер и выберите «Запуск от имени администратора» |
| **Оверлей Steam отключен** | Отсутствуют файлы интеграции со Steam | Проверьте наличие файла nvse_steam_loader.dll в корневой папке |
