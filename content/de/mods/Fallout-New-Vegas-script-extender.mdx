Um Ihr System auf den **Fallout New Vegas Script Extender** vorzubereiten, müssen Sie drei primäre Abhängigkeiten herunterladen und installieren. Diese stellen sicher, dass Ihr Windows-Betriebssystem den vom Extender injizierten benutzerdefinierten Code ausführen kann.

| Tool | Zweck | Quelltyp |
| --- | --- | --- |
| **7-Zip** | Entpackt komprimierte .7z- und .zip-Mod-Archive | Dienstprogramm |
| **Visual C++ Redistributable** | Stellt Laufzeitbibliotheken bereit, die für das moderne xNVSE erforderlich sind | Systembibliothek |
| **4GB-Patch** | Ermöglicht dem Spiel, 4 GB RAM anstelle von 2 GB zu nutzen | Ausführbarer Patch |

Laden Sie zuerst 7-Zip herunter und installieren Sie es, um das Entpacken von Dateien zu ermöglichen. Laden Sie als Nächstes die neuesten Laufzeitbibliotheken herunter. Viele Benutzer erleben sofortige Abstürze, weil ihnen die erforderlichen Microsoft-Bibliotheken fehlen.

<Callout type="warning" title="Systembibliothek-Anforderung">
Sie müssen das [Microsoft Visual C++ Redistributable](https://learn.microsoft.com/en-us/cpp/windows/latest-supported-vc-redist) installieren, bevor Sie den Script Extender starten. Ohne dieses Paket kann der Game Loader die benutzerdefinierten Skriptbibliotheken nicht in den Spielprozess injizieren, was zu einem Absturz ohne Fehlermeldung beim Start führt.
</Callout>

Laden Sie schließlich das 4GB-Patch-Dienstprogramm herunter. Moderne Betriebssysteme beschränken ältere 32-Bit-Anwendungen auf 2 GB virtuellen Speicher. Der Patch modifiziert die ausführbare Datei des Spiels, um 4 GB Adressraum zu verwalten, was Abstürze aufgrund von Speichermangel verhindert, wenn hochauflösende Texturen oder schwere Skript-Mods verwendet werden.

</Section>

<Section icon={Target} title="Schritt-für-Schritt-Einrichtung des Fallout New Vegas Script Extenders">

Sobald die erforderlichen Dateien bereitstehen, können Sie mit der manuellen Installation des **Fallout New Vegas Script Extenders** fortfahren. Der Installationsprozess beinhaltet das Platzieren der Dateien direkt im Stammverzeichnis Ihrer Spielinstallation und nicht im Data-Ordner, in dem sich Standard-Mods befinden.

<Steps>
  <Step num="1" title="Spielverzeichnis finden">
    Öffnen Sie Ihren Game-Launcher (z. B. Steam). Klicken Sie mit der rechten Maustaste auf das Spiel in Ihrer Bibliothek, wählen Sie „Verwalten“ und klicken Sie auf „Lokale Dateien durchsuchen“. Dadurch wird der Stammordner geöffnet, der die Hauptdateien des Spiels enthält.
  </Step>
  <Step num="2" title="NVSE-Dateien entpacken">
    Öffnen Sie das heruntergeladene Archiv des Script Extenders mit 7-Zip. Wählen Sie alle Dateien im Archiv aus (insbesondere die Loader- und DLL-Dateien) und ziehen Sie sie direkt in das Stammverzeichnis.
  </Step>
  <Step num="3" title="4GB-Patch anwenden">
    Laden Sie das Dienstprogramm [FNV 4GB Patch](https://www.nexusmods.com/newvegas/mods/62552) herunter. Entpacken Sie die ausführbare Datei in dasselbe Stammverzeichnis. Klicken Sie mit der rechten Maustaste auf die Patch-Datei, wählen Sie „Als Administrator ausführen“ und drücken Sie eine beliebige Taste, wenn die Konsole bestätigt, dass der Patch abgeschlossen ist.
  </Step>
  <Step num="4" title="Launcher konfigurieren">
    Der 4GB-Patch konfiguriert das Spiel automatisch so, dass der Script Extender beim Starten der Standard-EXE-Datei geladen wird. Sie müssen die alte Loader-Anwendung nicht verwenden, um das Spiel zu starten.
  </Step>
</Steps>

Es ist wichtig, die Dateien im richtigen Verzeichnis zu platzieren. Beachten Sie die folgende Zuordnung der Verzeichnisstruktur, um sicherzustellen, dass Ihre Dateien korrekt platziert sind.

| Dateiname | Zielverzeichnis | Dateityp |
| --- | --- | --- |
| **nvse_loader.exe** | Stammordner (wo sich FalloutNV.exe befindet) | Ausführbare Datei |
| **nvse_1_4.dll** | Stammordner (wo sich FalloutNV.exe befindet) | Bibliothek |
| **nvse_steam_loader.dll** | Stammordner (wo sich FalloutNV.exe befindet) | Bibliothek |
| **FNVPatch.exe** | Stammordner (wo sich FalloutNV.exe befindet) | Ausführbarer Patcher |

<Callout type="info" title="Verzeichnisüberprüfung">
Stellen Sie sicher, dass die Dateien des Script Extenders im Hauptverzeichnis platziert sind, das die ausführbare Datei des Spiels enthält, und nicht im Data-Ordner. Das Platzieren dieser Dateien im Data-Ordner verhindert das Laden des Script Extenders.
</Callout>

Wenn Sie die GOG-Version des Spiels verwenden, ist die ausführbare Datei bereits so gepatcht, dass sie standardmäßig 4 GB RAM unterstützt. Es wird jedoch empfohlen, das manuelle Patcher-Dienstprogramm dennoch auszuführen, um sicherzustellen, dass sich der Loader des Script Extenders korrekt in die Startroutine einklinkt.

</Section>

<Section icon={Sparkles} title="Überprüfung der Installation und Einrichtung des Mod-Managers">

Nach dem Kopieren der Dateien müssen Sie überprüfen, ob der Script Extender in der Spiel-Engine korrekt ausgeführt wird. Sie können dies überprüfen, indem Sie die Entwicklerkonsole öffnen. Darüber hinaus sollten Sie Ihren Mod-Manager so konfigurieren, dass er den Script Extender erkennt.

Um die Installation zu überprüfen, starten Sie das Spiel und drücken Sie die Tilde-Taste (~) auf Ihrer Tastatur, um die Entwicklerkonsole zu öffnen. Geben Sie den Befehl `GetNVSEVersion` ein und drücken Sie die Eingabetaste. Die Konsole sollte die aktive Versionsnummer zurückgeben. Wenn ein Fehler ausgegeben wird, dass der Befehl nicht gefunden wurde, sind die Dateien nicht im richtigen Verzeichnis installiert.

Für die Verwaltung von Mods wird Mod Organizer 2 empfohlen. Er verwendet ein virtuelles Dateisystem, das Ihr Spielverzeichnis sauber hält und verhindert, dass Mods die Dateien des Basisspiels dauerhaft überschreiben.

<Callout type="tip" title="Konfiguration von Mod Organizer 2">
Wenn Sie Mod Organizer 2 verwenden, starten Sie das Spiel immer über die Option für die gepatchte ausführbare Datei, die im Manager konfiguriert ist. Stellen Sie sicher, dass der Programmpfad direkt auf die gepatchte ausführbare Datei im Stammverzeichnis verweist, um alle virtualisierten Mods und Skripterweiterungen zu laden.
</Callout>

Wenn Sie den Script Extender in Zukunft aktualisieren möchten, laden Sie die neueste Version aus dem [xNVSE Github-Repository](https://github.com/xNVSE/xnvse) herunter und überschreiben Sie die vorhandenen Dateien in Ihrem Stammverzeichnis. Dadurch werden die Bibliotheken aktualisiert, ohne dass sich dies auf Ihre installierten Mods oder Speicherstände auswirkt.

</Section>

<Section icon={Swords} title="Fehlerbehebung bei häufigen NVSE-Fehlern">

Selbst bei sorgfältiger Installation können Konfigurationskonflikte oder fehlende Systemupdates Fehler verursachen. Zu verstehen, wie man diese Probleme diagnostiziert, hilft, die Spielstabilität aufrechtzuerhalten.

| Fehlersymptom | Häufige Ursache | Lösung |
| --- | --- | --- |
| **Absturz beim Start** | Fehlende Visual C++ Redistributables | Installieren Sie die neuesten x86- und x64-Laufzeitumgebungen |
| **Befehl nicht gefunden** | Dateien im falschen Ordner platziert | Verschieben Sie die Dateien von „Data“ in den Stammordner |
| **Patcher startet nicht** | Eingeschränkte Administratorrechte | Klicken Sie mit der rechten Maustaste auf den Patcher und wählen Sie „Als Administrator ausführen“ |
| **Steam-Overlay deaktiviert** | Dateien für die Steam-Integration fehlen | Überprüfen Sie, ob sich die Steam-Loader-DLL-Datei im Stammordner befindet |

Um sicherzustellen, dass Ihr Spiel vollständig optimiert und stabil ist, gehen Sie diese Checkliste durch, bevor Sie einen neuen Spieldurchlauf starten.
